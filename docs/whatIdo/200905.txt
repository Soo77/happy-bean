스프링 프레임워크
- 자바 엔터프라이즈 개발을 편하게 해주는 오픈소스 경량급 애플리케이션 프레임워크.
- 자바 플랫폼을 위한 오픈 소스 애플리케이션 프레임워크

애플리케이션 프레임워크? 
- 특정 계층이나, 기술, 업무 분야에 국한되지 않고 애플리케이션의 전 영역을 포괄하는 범용적인 프레임워크

@ComponentScan 어노테이션이란?
- Class들을 자동으로 Scan 하여 Bean으로 등록해주는 역할

@Bean 정의?
- 데이터를 표현하는 것을 목적으로 하는 자바 클래스.
- 반복적으로 코드를 따로 작성하여 재사용하기 위해 만들어진 클래스

빈을 만드는 방법에는 크게 두가지가 있다.
- Component Scanning
- 빈 설정파일에 직접 빈을 등록

Component Scan
- @ComponentScan 어노테이션과 @Component 어노테이션을 사용해서 빈을 등록하도록 하는 방법.
- @ComponentScan 어노테이션은 어느지점부터 컴포넌트를 찾으라고 알려주는 역할을 하고 @Component는 실제로 찾아서 빈으로 등록할 클래스를 의미한다.

Spring Benas
- 자바 객체이다. 일반 자바의 객체와 같은 속성을 지닌다.
- 스프링 컨테이너에 의해 자바 객체가 만들어지고, 관리되기 때문에 Spring Beans 라고 부른다.

@EnableWebMvc 왜 쓰는지 모르겠다
- 이 어노테이션을 사용하면 스프링프레임워크에서 여러 Config값을 알아서 설정해준다고 “한다.” 나는 이해못하는 중이고 그렇다고 한다.
- web mvc를 이용하는데 있어서 Spring Container가 가져야할 기본적인 mvc를 구축할 수 있는 configuration 환경을 제공해준다고 “한다.” zzz bb
- 이 어노테이션을 선언하면 
1. XML based configuration (configuration이 뭐지..? nn)
2. Annotation base configuration 
위 두가지를 지원한다고 “한다.” 즉, 이 어노테이션을 붙여줘야지만 @Configuration 어노테이션을 제대로 적용하게 된다고 “한다.”

@Configuration
- 이 어노테이션은 스프링 IOC container 에게 해당 클래스를 Bean 구성 Class 임을 알려주는 것이다.

@Bean vs @Component
- 둘다 스프링 컨테이너에 Bean을 등록하도록 하는 메타데이터를 기입하는 어노테이션이다.
@Bean의 경우 개발자가 직접 제어가 불가능한 외부 라이브러리들을 Bean으로 만들려할 때 사용된다.
@Component 의 경우 개발자가 직접 작성한 Class를 Bean으로 등록하기 위한 어노테이션이다.

@PropertySource가 뭐지?
- 스프링에서 런타임시 프로퍼티 값을 가져오기 위해서 사용하는 것 두가지가 있다.
1. PropertySource
2. 스프링 표현 언어 SqEL(spring Expression Language)
요 두 개.

@PropertySource("classpath:com/soo/conf/jdbc.properties")
public class AppConfig {
}
- @PropertySource 어노테이션에 jdbc.properties의 위치를 넣어주면, Environment 객체에 프로퍼티값이 자동으로 주입된다.
